## Пользовательский результат

Сам по себе технический переход не должен резко перекрасить продукт. Пользователь
получает более согласованные карточки, таблицы, статусы и действия, сохранение
выбора без полной перерисовки страницы, более предсказуемые ошибки и возможность
быстрее исправлять один общий компонент во всех мигрированных модулях.

На переходном этапе соседние модули могут выглядеть по-разному. Это допустимо,
если каждый новый вертикальный сценарий использует общий UI-контракт, сравнивается
с legacy на одинаковых данных и имеет мгновенный rollback. Сначала унифицируется
компонентный язык, затем — оставшиеся экраны; ожидание полной визуальной
унификации legacy до старта лишь законсервировало бы расхождения.

## Что ещё не завершено

- authenticated Pilot read-only QA оставшихся islands и отдельное решение о default-on для уже принятых срезов;
- перенос остальных write-команд: справочники, назначения, факт и публикация;
  create/edit/delete Номенклатуры уже реализованы как один отдельный default-off
  evaluation-срез;
- PIN/сессия и write-команды авторизации; React уже покрывает только безопасный
  выбор отдела, участка и сотрудника, а Gantt, Specifications 2.0 и Contour
  Admin также production-интегрированы read-only default-off;
- удаление legacy возможно только после функциональной parity, Pilot-приёмки и
  проверенного rollback для каждого сценария.
