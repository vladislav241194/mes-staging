# Figma-first: модуль Гант

Этот каталог содержит независимый дизайн-прототип модуля Гант. Он не подключен к рабочему приложению и не меняет бизнес-логику.

## Файлы

- `index.html` — интерактивно-просмотровой макет с тремя фреймами.
- `styles.css` — визуальные токены, layout и компоненты макета.
- `figma-import-overview.svg` — упрощенная SVG-схема для импорта в Figma.

## Фреймы

- Desktop: `1440x900`
- Laptop: `1280x720`
- Mobile: `390x844`

## Концепция

Центральная модель экрана:

```text
Маршрутная карта
└── Операции маршрутной карты
    └── Слоты операций во времени
```

Подразделение не является основной строкой Ганта. Оно отображается как вторичная информация у операции и используется для календаря рабочего/нерабочего времени.

## Основные зоны

- Главный sidebar системы.
- Верхняя панель модуля.
- Toolbar с периодом, масштабом, zoom, режимом содержимого колбаски, поиском и фильтрами.
- Левая фиксированная колонка диаграммы.
- Временная сетка.
- Строки маршрутной карты и операций.
- Колбаски операций.
- Нерабочие зоны.
- Стрелки зависимостей.
- Right Detail Drawer операции.
- Mobile-представление в виде списка операций с мини-таймлайном.

## Цветовая логика

- Обычные операции: нейтральный серый.
- Активная/выбранная операция: мягкий синий.
- Операции склада/готовой продукции: зеленый.
- Нерабочее время: серые зоны.
- Предупреждения: янтарный акцент, если потребуется отдельное состояние.

## Правила UX

- В строках Ганта должны быть операции, а не подразделения.
- Внутри операции показывать название и количество, если хватает ширины.
- При нехватке ширины переходить к compact/tiny-отображению.
- Колбаски не должны выходить за границы строки.
- Левая колонка должна оставаться стабильной и не наезжать на сетку.
- Временная сетка должна поддерживать шаг 15 минут в часовом масштабе.
- Нерабочее время строится только по графику подразделения операции.
- Стрелки зависимостей не должны закрывать текст внутри колбасок.

## Что переносить в рабочий интерфейс

1. Структуру toolbar.
2. Строку маршрутной карты как group header.
3. Строки операций как основные строки диаграммы.
4. Compact-режим колбасок.
5. Right Detail Drawer как основной способ просмотра операции.
6. Mobile-представление не как уменьшенный Гант, а как список операций с мини-таймлайном.
