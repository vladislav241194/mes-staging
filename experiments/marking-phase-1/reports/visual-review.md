# Визуальная оценка рабочего места «Маркировка»

Дата проверки: 20.07.2026
Ветка: `codex/frontend-react-migration`

## Оценочный срез

Прототип показывает отдельный модуль сотрудника участка маркировки, а не универсальный рабочий стол для всех отделов.

- два явно помеченных MOCK-задания;
- выбор задания в компактной очереди;
- подготовка количества мультипликаций и плат;
- расчёт мастер-кодов, индивидуальных кодов и этикеток;
- тестовая частичная печать с отдельным подтверждением результата;
- таблица комплектов с постраничным выводом;
- добавление объёма, завершение, передача и read-only проверка кода;
- полный сброс состояния в памяти вкладки.

SMT и склад показаны только как отключённые самостоятельные модули. Их функциональность не подменяется рабочим местом маркировки.

## Результат критики компоновки

Удалена верхняя KPI-полоса: она повторяла количество и статусы из очереди. Из карточек очереди удалены план и расчёт этикеток: эти данные теперь находятся только в рабочей области выбранного задания. Отдельный блок фактов также удалён, потому что дублировал поля подготовки партии.

Итоговая иерархия:

1. очередь определяет, с каким заданием работает сотрудник;
2. заголовок рабочей области подтверждает выбранный контекст;
3. блок подготовки содержит единственный источник количеств и расчёта;
4. вкладки разделяют комплекты, партии печати и историю.

## Визуальная проверка

- Desktop `1440 × 1000`: очередь 340 px, рабочая область 773 px, горизонтального overflow нет.
- Mobile `390 × 844`: очередь вертикальная, обе карточки доступны без горизонтального скролла; рабочая область 355 px; горизонтального overflow страницы нет.
- Логотип загружается из `assets/brand/mes_logo_high_quality.svg`, естественный размер `512 × 512`; дополнительной цветной подложки и внутренних CSS-отступов нет.
- Используется существующий шрифт Onest из `assets/fonts/onest/Onest-Variable.woff2`.
- Ошибок загрузки логотипа после настройки Vite allow-list нет.

## Функциональная проверка

- `npm run test`: 4/4 model-теста.
- `npm run build`: TypeScript и Vite собираются.
- Частичная MOCK-печать: 10 комплектов / 210 этикеток; после подтверждения 10 строк получили статус успешной печати.
- «Сбросить MOCK» доступна и на desktop, и на mobile; возвращает `0 из 100` напечатанных комплектов.
- В исходниках нет `fetch`, `XMLHttpRequest`, `WebSocket`, `localStorage` или `sessionStorage`.

## Что сознательно не реализовано

Прототип не подключает реальные назначения, права, аудит, БД, API, оборудование, камеру и жизненный цикл дефектов/ремонтов. Это границы последующей продуктовой и архитектурной фазы, а не готовность к Pilot.
