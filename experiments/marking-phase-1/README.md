# MES Line · рабочее место «Маркировка» · MOCK-прототип

Изолированный прототип отдельного рабочего места маркировки на React + TypeScript. Он предназначен только для оценки компоновки и короткого интерактивного сценария.

## Запуск

```bash
cd /Users/vladislav/Documents/Codex/2026-05-30/mes-frontend-react/experiments/marking-phase-1
npm ci
npm run dev -- --port 4179
```

Открыть `http://127.0.0.1:4179/`.

Проверка:

```bash
npm run check
```

## Границы и MOCK-данные

- Все задания, СЗН, сотрудники, коды, партии и события созданы в `src/testData.ts` и помечены в интерфейсе как `MOCK`.
- Состояние существует только в памяти открытой вкладки. `localStorage`, `sessionStorage`, API, PostgreSQL и runtime MES не используются.
- Перезагрузка страницы или кнопка «Сбросить MOCK» возвращает исходное состояние.
- Печать проходит только через `testPrintAdapter`; физический принтер и сетевые сервисы не вызываются.
- Сканер работает только на чтение; камера — визуальная имитация без доступа к устройству.
- Pilot, legacy runtime и feature flags не изменены.

## Почему такая компоновка

Рассматривались три варианта:

1. Общая KPI-сводка + очередь + карточка задания. Отклонён: сводка повторяла данные очереди и рабочей области.
2. Горизонтальная доска этапов. Отклонена: для сотрудника маркировки этапы стабильны, а карточки занимают место без добавления информации.
3. Очередь слева + единая рабочая область справа. Выбрана: сотрудник сначала выбирает задание, затем выполняет действия в одном контексте без повторяющихся KPI.

На телефоне очередь становится вертикальной; рабочая область идёт следом. Это сохраняет полный контекст двух MOCK-заданий без обрезанных карточек.
